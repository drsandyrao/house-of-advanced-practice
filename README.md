# The House of Advanced Practice

An accessible static educational webpage for graduate social work clinical practice. The site is an interactive memory palace with a game-like map, clickable rooms, professional standards, and a browser-saved Clinical Decision Notebook.

This project is plain HTML, CSS, and JavaScript. It does not require a build step, package manager, database, or server-side code, so it can be hosted directly on GitHub Pages.

## Where To Edit Educational Content

Most course content lives in `assets/data/site-data.js`.

Edit this file to update:

- room names
- professional guide names
- clinical questions
- room explanations
- practice prompts
- scenario questions
- professional standards links
- notebook summary headings
- map labels and colours

The page reads this file when it loads, so routine content edits should not require changing `index.html`.

## Where To Edit The Guide Slide Deck

The guide character slide deck lives in `slides/characters/`.

Edit `slides/characters/slides.js` to update:

- slide titles and subtitles
- guide names
- room names
- core questions
- teaching points
- "When this guide speaks..." phrases
- avatar accent colours and symbols

The slide layout and visual styling are in `slides/characters/index.html` and `slides/characters/styles.css`.

## How To Change Colours

For room and map colours, edit the `theme` and `mapSites` sections in `assets/data/site-data.js`.

For global interface colours, edit the CSS variables near the top of `assets/css/styles.css`:

```css
:root {
  --cream: #f8f6f2;
  --paper: #fdfbf7;
  --charcoal: #353535;
  --forest: #2f5d50;
  --bronze: #b58a57;
}
```

Keep colour contrast high enough for readability, especially for text, buttons, links, and focus outlines.

## How To Add A Room

1. Open `assets/data/site-data.js`.
2. Copy an existing object in the `rooms` array.
3. Give the new room a unique `id`.
4. Add the matching room text, prompts, scenario, standards, and summary category.
5. Copy an existing object in the `mapSites` array.
6. Set `roomId` to the same unique `id`.
7. Adjust the SVG `shape`, `hit`, `label`, `level`, and plaque coordinates so the new room appears in the map.
8. If the room needs a new visual symbol, add a matching icon drawing in `assets/js/app.js` inside the `renderIcon()` function.

## How To Change Icons

The current map symbols are lightweight SVG drawings generated by `assets/js/app.js`.

To change which icon a room uses, edit its `icon` value in `assets/data/site-data.js`.

To change the drawing itself, edit the matching icon template in the `renderIcon()` function in `assets/js/app.js`.

Future image artwork can be placed in `assets/images/`; future icon files can be placed in `assets/icons/`.

## Notebook Notes

The Clinical Decision Notebook saves notes in the visitor's browser using `localStorage`.

This means:

- notes stay on the same device and browser
- notes are not sent to a server
- clearing browser data may remove saved notes
- changing the `storageKey` in `assets/data/site-data.js` starts a new notebook storage space

## Run Locally

You can open `index.html` directly in a browser. For the closest GitHub Pages test, serve the folder locally:

```bash
python3 -m http.server 8123
```

Then open:

```text
http://127.0.0.1:8123/
```

## Deploy To GitHub Pages

1. Create a new GitHub repository.
2. Upload the contents of this folder to the root of the repository. `index.html` should be at the top level.
3. Commit the files to the `main` branch.
4. In GitHub, open **Settings**.
5. Open **Pages**.
6. Under **Build and deployment**, choose **Deploy from a branch**.
7. Select branch `main` and folder `/root`.
8. Save.
9. Wait for GitHub to publish the site, then open the Pages URL shown in the settings screen.

## Update The Site Later

1. Edit `assets/data/site-data.js` for content changes.
2. Edit `slides/characters/slides.js` for guide slide deck changes.
3. Edit `assets/css/styles.css` or `slides/characters/styles.css` for visual changes.
4. Commit the changed files to GitHub.
5. GitHub Pages will redeploy automatically.

## Optional Custom Domain

1. Buy or configure a domain with your DNS provider.
2. In GitHub Pages settings, enter the custom domain.
3. Follow GitHub's DNS instructions for either a `CNAME` record or apex domain records.
4. When GitHub confirms the domain, enable HTTPS.

## Troubleshooting

- **404 page:** Confirm `index.html` is in the repository root and GitHub Pages is set to `main` and `/root`.
- **Old version still appears:** Hard refresh the browser or clear cache. GitHub Pages may also take a few minutes to update.
- **Fonts do not load:** Confirm all font files are present in `assets/fonts/` and the CSS paths still begin with `../fonts/`.
- **Blank page:** Open the browser console and check that `assets/data/site-data.js` and `assets/js/app.js` are loading.
- **Notes disappeared:** Notes are stored locally in the browser. They may disappear if browser data is cleared or if the notebook `storageKey` changes.
- **A room does not open:** Confirm the room `id` in the `rooms` array matches the `roomId` in the `mapSites` array.
